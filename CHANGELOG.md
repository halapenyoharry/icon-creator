2026-04-26: Compared two approaches to AI-generated icons/logos:
1. First approach: Analyze repo keywords/themes, map to visual motifs (procedural, generative, AI, exploration), and generate an abstract SVG using programmatic art libraries. Driven by semantic analysis, not project structure.
2. Second approach (inspired by ~/Projects/collection): Use composable, parameterized generative art layers (particles, crystalline, growth, etc.), encode project data into parameters, and blend layers for a unique, data-driven image. More modular, compositional, and reflective of project structure.
Key difference: The second approach leverages modular generative logic and project data for deeper, more meaningful visual identity, while the first is more generic and theme-based.
